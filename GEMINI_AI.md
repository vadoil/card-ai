# Подключение и «обучение» Gemini для Stock AI

Коротко и по делу: что реально означает «обучить» Gemini, что дешевле, что дороже,
и какой план под наш кейс (генерация карточек маркетплейсов, анализ ниши, контент-план).

---

## 1. Что значит «обучить модель» — 3 уровня (от дешёвого к дорогому)

### Уровень 1. **Prompt engineering + системный промпт** (≈ бесплатно)
Никакого обучения. Мы просто даём модели жёсткую инструкцию + примеры в каждом запросе.
- Стоимость: только токены запроса.
- Скорость внедрения: часы.
- Подходит для: 80% задач Stock AI (генерация title/описания/буллетов, рерайт под WB/Ozon).
- Минус: каждый раз шлём «учебник» в промпте → длинный input.

### Уровень 2. **RAG (Retrieval-Augmented Generation)** (дёшево)
Храним свою базу знаний (правила маркетплейсов, удачные карточки, шаблоны, требования по нишам)
в векторной БД. На каждый запрос достаём 3–10 релевантных кусков и подкладываем модели.
- Стоимость: эмбеддинги (копейки) + чуть больший input.
- Подходит для: «анализ ниши», «контент-план под категорию», «как сделать топ выручки».
- Что нужно: векторная БД (pgvector в Postgres / Qdrant / Chroma) + embeddings от Gemini
  (`text-embedding-004` — практически бесплатные).

### Уровень 3. **Fine-tuning (Supervised Tuning)** Gemini (дороже, но не критично)
Берём 100–1000+ пар «вход → идеальный ответ» и дообучаем модель.
- Доступно на `gemini-1.5-flash-001-tuning` через Vertex AI / AI Studio.
- Стоимость обучения: разово, зависит от размера датасета (обычно $10–$100).
- Inference тюненой модели стоит **столько же**, сколько базовой Flash.
- Подходит, когда нужно стабильно держать наш стиль карточек, тон, структуру.
- **Минимум: 100 примеров, рекомендуется 500+.**

> Полноценный «train from scratch» / LoRA весов Gemini — недоступно, модель закрытая.
> Так что наш максимум — supervised fine-tuning через Google.

## 2. Какую модель брать (цены на 2025)

| Модель                  | Вход $/1M ток | Выход $/1M ток | Куда                                            |
|-------------------------|---------------|----------------|-------------------------------------------------|
| `gemini-2.0-flash`      | ~$0.10        | ~$0.40         | **Дефолт.** Карточки, заголовки, описания.      |
| `gemini-2.0-flash-lite` | ~$0.075       | ~$0.30         | Массовая генерация, батчи, рерайт.              |
| `gemini-1.5-flash`      | ~$0.075       | ~$0.30         | Совместимость, fine-tuning.                     |
| `gemini-2.5-pro`        | ~$1.25        | ~$10           | Только сложный анализ ниши/стратегия выручки.   |
| `text-embedding-004`    | ~$0.00 (free tier щедрый) | —  | Векторизация базы знаний для RAG.               |

**Правило экономии для Stock AI:**
1. Генерация карточек → `gemini-2.0-flash-lite` или `flash`.
2. Анализ ниши, «как поднять выручку» → `gemini-2.5-pro`, но **только финальный шаг**;
   подготовку данных делает flash.
3. Эмбеддинги для RAG → `text-embedding-004`.

## 4. Как собрать датасет для fine-tuning (дёшево и руками)

Формат файла — JSONL, по строке на пример:

```jsonl
{"input_text":"Ниша: детские термокружки. Сделай title для WB.","output_text":"Термокружка детская 350 мл с трубочкой, не проливается — для школы и сада"}
{"input_text":"Ниша: смарт-часы мужские. Дай 5 буллетов для Ozon.","output_text":"• Звонки по Bluetooth 5.3\n• Пульс и SpO2 24/7\n• ..."}
```

Источники примеров:
- Свои лучшие карточки (если есть).
- Топ-10 карточек по выдаче WB/Ozon в каждой нише (парсим title + описание).
- Ручная редактура GPT-выдачи → сохраняем «эталон» как `output_text`.

Минимум 100 строк → уже даст эффект. 500+ → стабильный стиль.

Загрузка:
- Google AI Studio → Tuning → New tuned model → загрузить JSONL → выбрать
  base model `gemini-1.5-flash-001-tuning` → Train. По итогу получаем
  `tunedModels/stock-ai-cards-v1` — его и зовём в коде.

## 5. Как звать Gemini из нашего бэка (TanStack server function)

> Ключ `GEMINI_API_KEY` уже будет в `.env` (см. `DEPLOY.md` §8).

```ts
// src/lib/gemini.functions.ts
import { createServerFn } from "@tanstack/react-start";
import { GoogleGenerativeAI } from "@google/generative-ai";

export const generateCard = createServerFn({ method: "POST" })
  .inputValidator((d: { niche: string; marketplace: "wb" | "ozon" }) => d)
  .handler(async ({ data }) => {
    const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);
    const model = genAI.getGenerativeModel({
      model: "gemini-2.0-flash",
      systemInstruction:
        "Ты — редактор карточек для маркетплейсов РФ. Пиши кратко, по правилам " +
        data.marketplace.toUpperCase() + ". Без воды, без эмодзи в title.",
    });
    const res = await model.generateContent(
      `Сделай title + 5 буллетов + описание (до 1000 символов). Ниша: ${data.niche}`
    );
    return { text: res.response.text() };
  });
```

Установить:
```bash
bun add @google/generative-ai
```

## 6. Дешёвый план запуска ИИ в Stock AI (по шагам)

1. **Неделя 1.** Prompt + system instruction на `gemini-2.0-flash-lite`. Кешируем повторные запросы.
2. **Неделя 2.** Поднимаем `pgvector` в Lovable Cloud, заливаем 200–500 эталонных карточек,
   считаем эмбеддинги `text-embedding-004`. RAG для генерации.
3. **Неделя 3.** Собираем JSONL из лучших ответов (с правкой), готовим dataset.
4. **Неделя 4.** Fine-tune `gemini-1.5-flash-001-tuning` → переключаем «Топ»-тариф на tuned-модель.
5. **Позже.** Для Pro-анализа ниши — отдельный endpoint на `gemini-2.5-pro`,
   вызывается редко (1–2 раза в день на пользователя) → бюджет под контролем.

## 7. Контроль расходов

- Включить лимиты на проект в Google AI Studio (Budget alert: $10 → $50 → $200).
- На каждый вызов логировать `inputTokens / outputTokens` (есть в ответе SDK).
- Для массовых генераций — **Batch API Gemini** (дешевле на ~50%).
- Кешировать «системный промпт + RAG-контекст» через **Context Caching** —
  у Gemini это отдельная фича, экономит до 75% input-стоимости при повторных запросах.

---

TL;DR:
- Начни с **flash-lite + RAG** — этого хватит на 80% задач за копейки.
- **Fine-tune** делай только когда нужен стабильный стиль и у тебя ≥ 200 примеров.
- **Pro-модель** — только для редких «умных» сценариев (анализ ниши / стратегия).
